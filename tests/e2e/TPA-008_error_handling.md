# TPA-008 エラーハンドリング

- 担当: Dev-FE, Dev-BE, QA
- 目的: 5xx/ネットワーク失敗を1000ms以内に通知し、再試行で成功

## 前提
- テスト用に 500 を返すスイッチ（envやクエリ）を用意

## 操作
1) 500擬似を有効化した状態で検索
2) エラーメッセージ表示を確認
3) `再試行` を押下し、同一リクエストを再送
4) 正常化して成功を確認
5) 400エラー（VALIDATION_ERROR）はフォームの各フィールドにマッピングされ、入力は保持されることを別途確認

## 期待結果
- 1000ms以内に「取得に失敗しました。再試行してください。」が表示
- 再試行で成功
- 400 reasonsは各フィールドの `role=alert` 等に反映、既入力値は保持
