# TPA-002 即時バリデーション（理由付き）

- 担当: Dev-FE, Dev-BE(メッセージ整合), QA
- 目的: blur/submitで100ms以内にエラー表示。エラー時API呼び出しは0回

## 前提
- バリデーション文言: 
  - 空destination → 「行き先は必須です」
  - start_date未入力 → 「開始日は必須です」
  - end_date未入力 → 「終了日は必須です」
  - start_date > end_date → 「終了日は開始日以降にしてください」
  - 期間>30日 → 「期間は30日以内にしてください」

## 操作
1) `/` を表示
2) `行き先` を空のまま blur → エラーを確認
3) `開始日`/`終了日` を空のまま blur → 各エラーを確認
4) `開始日=2025-10-10`, `終了日=2025-10-05` を入力→ blur

## 期待結果
- 各ケースで100ms以内にエラーテキスト（`role=alert`）が表示
- 上記(4)ではエラー文言が表示され、ネットワークリクエストは0件

## 観測
- ブラウザのnetwork記録を用い、エラーケースでは `POST /api/itinerary` / `/api/plan` が未送信であることを検証
