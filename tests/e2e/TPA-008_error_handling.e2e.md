# TPA-008 エラーハンドリング E2E雛形

テスト名: E2E-TPA-008-01 5xx/ネットワーク失敗で 1000ms 以内にメッセージ、再試行で成功

- 前提:
  - BE を 500 擬似応答できるよう設定（ルート、もしくはテスト用フラグ）。

- 操作:
  1) 500 を返す状態でフォーム送信。
  2) エラーメッセージ表示までの時間を計測。
  3) 「再試行」をクリックし、同一リクエストを再送。

- 期待結果:
  - 「取得に失敗しました。再試行してください。」が 1000ms 以内に表示。
  - 再試行成功で結果が表示。
  - 400 の `reasons` はフォーム各フィールドにマッピングされ、入力値は保持される（別ケースで検証）。
